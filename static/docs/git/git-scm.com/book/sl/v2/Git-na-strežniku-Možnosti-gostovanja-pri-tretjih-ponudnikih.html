4.9 Git na strežniku - Možnosti gostovanja pri tretjih ponudnikih

Možnosti gostovanja pri tretjih ponudnikih

Če ne želite iti skozi celotno opravilo, ki je potrebno za nastavitev vašega lastnega strežnika Git, imate nekaj možnosti za gostovanje svojih projektov Git na zunanjih namenskih straneh za gostovanje. To ponuja številne prednosti: gostujoča stran se v splošnem hitro postavi, enostavnejše je začeti projekte in ni potrebnega nobenega vzdrževanja strežnikov ali spremljanja. Tudi če nastavite in poženete svoj lastni strežnik interno, boste morda še vedno želeli uporabiti javno gostujočo stran za svojo odprto kodo — v splošnem je za odprtokodno skupnost enostavnejše, da vas najde in vam pri tem pomaga.

Te dni imate na izbiro veliko število možnosti gostovanja, vsaka z različnimi prednostmi in slabostmi. Da si ogledate posodobljen seznam, preverite stran GitHosting na glavnem wikiju Git na https://archive.kernel.org/oldwiki/git.wiki.kernel.org/index.php/GitHosting.html.

Podrobno bomo pokrili uporabo GitHuba v razdelku GitHub, saj je največji gostitelj Gita na voljo in morda boste morali tako ali tako sodelovati s projekti, ki so na njem gostujoči, vendar na izbiro je še ducat drugih, za katere vam ni treba nastavljati lastnega strežnika Git.
