1.7 Почеток - Getting Help

Getting Help

Ако некогаш ви е потребна помош при користење на Git, постојат два еквивалентни начини за да ја добиете помошта за сеопфатна рачна страница (manpage) за која било од командите Git:

$ git help <verb>
$ man git-<verb>

На пример, можете да ја добиете помошта за manpage за командата git config со извршување

$ git help config

Овие команди се убави затоа што можете да им пристапите насекаде, дури и офлајн. Ако деловните простори и оваа книга не се доволни и ви треба лична помош, можете да го пробате #git или` #github` каналот на Freenode IRC серверот (irc.freenode.net). Овие канали редовно се полнат со стотици луѓе кои се многу познавања за Git и често се подготвени да помогнат.

Покрај тоа, ако не ви е потребна целосна помош за manpage, но потребен е брз осврт на достапните опции за командата Git, можете да побарате поконзистентен ‘` help '’ излез со -h или --help опции, како во

$ git add -h
usage: git add [<options>] [--] <pathspec>...

    -n, --dry-run         dry run
    -v, --verbose         be verbose

    -i, --interactive     interactive picking
    -p, --patch           select hunks interactively
    -e, --edit            edit current diff and apply
    -f, --force           allow adding otherwise ignored files
    -u, --update          update tracked files
    -N, --intent-to-add   record only the fact that the path will be added later
    -A, --all             add changes from all tracked and untracked files
    --ignore-removal      ignore paths removed in the working tree (same as --no-all)
    --refresh             don't add, only refresh the index
    --ignore-errors       just skip files which cannot be added because of errors
    --ignore-missing      check if - even missing - files are ignored in dry run
    --chmod <(+/-)x>      override the executable bit of the listed files
